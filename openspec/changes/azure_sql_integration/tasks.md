# OpenSpec Task List: Azure SQL Persistence Integration

## Objective
Integrate an Azure SQL Database to persist `CRITICAL` and `WARNING` alerts generated by the `AnomalyValidationService`. 

## Prerequisites
- The local MCP server `@modelcontextprotocol/server-mssql` is active and connected to the target Azure SQL Database.

## Execution Tasks

- [x] **Task 1: Database Inspection (MCP)**
  - Use the MSSQL MCP tool to inspect the schema of the existing table named `AssetSensorReadings`.
  - Document the exact column names, data types, and primary key constraints.

- [x] **Task 2: Dependencies**
  - Open `build.gradle`.
  - Add `org.springframework.boot:spring-boot-starter-data-jpa`.
  - Add `com.microsoft.sqlserver:mssql-jdbc`.

- [x] **Task 3: Environment Configuration**
  - Open `src/main/resources/application.yml`.
  - Configure the `spring.datasource` properties using environment variables for security:
    - `url: ${AZURE_SQL_URL}`
    - `username: ${AZURE_SQL_USER}`
    - `password: ${AZURE_SQL_PASSWORD}`
    - `driver-class-name: com.microsoft.sqlserver.jdbc.SQLServerDriver`
  - Set `spring.jpa.hibernate.ddl-auto: validate` to ensure we do not overwrite the existing table schema.

- [x] **Task 4: JPA Entity Creation**
  - Create a new `@Entity` class named `AssetSensorReadingEntity`.
  - Map the fields *exactly* to the columns discovered via the MCP inspection in Task 1.
  - Create a corresponding `AssetSensorReadingRepository` interface that extends `JpaRepository`.

- [x] **Task 5: Service Integration**
  - Open `AnomalyValidationService.java`.
  - Inject the `AssetSensorReadingRepository`.
  - Update the `evaluateReading` method: If the threshold logic evaluates to `CRITICAL` or `WARNING`, map the `AnomalyReading` to an `AssetSensorReadingEntity` and call `repository.save()` *before* forwarding the alert to the Kafka template.

- [x] **Task 6: Build Verification**
  - Run `./gradlew build` to ensure the project compiles with the new JPA dependencies and logic.
